1. Introduction

The AgroDistribution™ ERP is a comprehensive, full-stack management solution designed for agro-chemical distribution centers. It streamlines complex operations including high-velocity inventory tracking, field-based distribution via truck loads, real-time invoicing, and multi-tier financial reconciliation.

Built with a "Local-First, Cloud-Ready" philosophy, the system utilizes a high-performance Node.js backend with an optimized SQLite (WAL-enabled) data engine and a premium Vanilla JavaScript frontend design system ("Agro-Green").

2. User Manual: Functional Workflows

2.1 Authentication & Security

Access the system via the secure login portal. Passwords are encrypted using the bcrypt algorithm. Inactive sessions are protected by server-side timeouts.

2.2 Inventory Management

The Product Registry allows for granular tracking of stock. Features include:

  • SKU & Barcode Support: Speed up billing and loading.
  • Unit Management: Track litres, kilograms, and carton units.
  • Supplier Tracking: Maintain records of procurement sources and area managers.

2.3 Sales (Point of Sale)

The Sales module supports Credit and Cash transactions. The system automatically memorizes the last discount percentage given to a specific customer for a specific product, ensuring pricing consistency in future bills.

2.4 Field Distribution (Truck Workflow)

Unique "Load & Reconcile" workflow:

  1. Morning Load: Assign stock to specific vehicles.
  2. Field Sales: Real-time stock deduction from the vehicle's inventory.
  3. Evening Reconcile: Calculate variance (Loaded - Sold - Returned).

3. Admin Features

3.1 User Permissions

The Admin Panel allows for Role-Based Access Control (RBAC). Admins can toggle permissions for sensitive actions like Clear Logs or Database Reset.

3.2 System Audit Center

The redesigned Audit module provides full visibility into CREATE, UPDATE, and DELETE events across all ledger tables, identified by timestamp and username.

4. Technical Documentation

4.1 Architecture Overview

The application follows a standard RESTful pattern:

Frontend (Browser) <--> Express.js Server <--> Repositories <--> SQLite (WAL)

4.2 Database Schema

Table Function
products Master inventory records and pricing.
customers Client profiles and ledger balances.
truck_loads Header records for distribution vehicles.
invoices Sales headers with tax and discount summaries.
audit_logs System-wide activity tracking.

4.3 Core API Endpoints

  • GET /api/dashboard/kpis: Real-time financial signals.
  • POST /api/distribution/load: Inventory allocation.
  • GET /api/reports/sales-analytics: Deep intelligence engine.

5. Deployment & Maintenance

Deployment requires a standard Node.js environment. Maintenance is simplified through automated midnight backups and integrated log purging modules.
